## Quick Summmary

Shrine is a free, open source game engine written in Crystal. Designed to be simple, because I am a new noob, it will feature a simple, functional API and a full 2-D/3-D feature set.
For more information on the project, check the other markdown files spread throughout.


#### What is Shrine?

Shrine is an engine that I am working on for creating an easy to use, GUI game engine with 2-D focus and 3-D functionality. It is licensed under MPLv2 with a soon to be unique runtime and buildtime library that is licensed under LGPLv3. Shrine attempts to be full-featured, including having a rebuild feature where the game engine can rebuild itself and all code with a 'click of a button'. The runtime should be lightweight, and parse a few languages including but not limited too (ever)
* Crystal (The First Class Citizen)
* Ruby (Via API)
* SWift (Via API)
and a soon to be unique scripting language called 
* Mist 
Think of Ruby and Mist as the dynamic options while Crystal and Swift are your static options. It is your choice which you want to use.
#### Can I release my own plug-ins, free or paid?

Yes, that is a planned featured. 

#### What about an editor?
The Editor at the moment will be a plain text editor, but an alternative to open an IDE via an exe is also available. 

#### Why not just use (insert FOSS game engine here)
Because this project exists to challenge myself with languages I love. There is absolutely nothing wrong with Godot, Bevy, FNA etc. But there is also no game engine for Crystal and these languages listed, so fuck it, I am making one myself.
#### What is the backend?
As of 12/17, only OpenGL dependencies are in the engine, but GTK, SDL and Raylib Core are also planned. SDL for hardware inputs, Raylib for a 'foundation' and GTK just to make it look pretty.
